# Support

Public open-source starter repositories under this organization may use their
own issue tracker or a documented public support channel as described in the
repository's README or CONTRIBUTING file.

This organization does **not** provide GitHub-based support for
private/proprietary Gideon services.

## Security concerns

Security vulnerabilities must be reported following the repository's
**SECURITY.md** policy. Do not disclose suspected vulnerabilities in public
issues, pull requests, or discussions.

## What not to include

Support requests must not include:

- credentials, tokens, API keys, or private keys
- customer or tenant data
- private logs or trace output
- production identifiers or internal hostnames
